## Coding Standards

Our team agrees to follow these coding standards:

- Use meaningful variable and function names.
- Follow Python naming conventions (snake_case).
- Use four spaces for indentation.
- Include comments when needed.
- Write docstrings for every function.
- Keep code neat and readable.

---

## Git Workflow

Our GitHub workflow will include:

- Create a branch before making changes.
- Commit changes frequently.
- Write meaningful commit messages.
- Push completed work to GitHub.
- Review changes before merging.

### Example Commit Messages

- Added Team Standards Document
- Added add_record() stub
- Updated documentation
- Fixed formatting
